Compatibility

vuSmartMaps supports the monitoring of your IBM AIX operating system's various components including hardware, software, network, and applications.

OS

vuSmartMaps supports following OS and their minimum versions:

OS Version
Version 11.2

Data Collection Method

vuSmartMaps collects health and performance data for IBM AIX server using VuNet's Java Based Health agent. This agent collects data based on the source configuration.

Prerequisites

Inputs for Configuring Data Source

To configure this data source, you will require following parameters:

Key Help Text
Host The IP Address/FQDN of the IBM AIX server. This field is the key to identify each server you add here.
Interval [in seconds] How frequently data is gathered. period should be between 60 seconds - 3000 seconds.

Firewall Requirement

To collect data from this O11ySource, please make sure that the following ports are opened:

Source IP Destination IP Destination Port Protocol Direction
IP address of the IBM AIX server vuSmartMaps Data Collector End Point 9092* TCP Inbound

*Before providing the firewall requirements, please update the port based on the customer environment.

Configuring the Target

VuNet's health agent should be installed and running in the IBM AIX server.

Configuration Steps

  1. Enable the IBM AIX O11ySource.
  2. Select the Sources tab and press the + button to add a new IBM AIX host to be monitored.
  3. Add the required details and then click on Next .
  4. The following packages will be available for download based on the OS:
    • vuHealthagent full installer package - Downloads the full vuHealthagent package with required configurations for a fresh installation
    • vuHealthagent config update package - Downloads the agent configuration package to update an existing vuHealthagent installation
  5. Download the agent installation or update package, then click Finish to close the data source window.

Agent: vuHealthagent

AIX

Fresh Agent Installation

  1. Extract the downloaded vuHealthagent package.
  2. Run the vuHealthagent installation script.
    ./install --quick --installpath <path> [--setupinit] [--enablessl]
    NOTE:
    • --setupinit and --enablessl are optional.
    • Use the --enablessl parameter to enable SSL communication between agent and vusmartmaps server.
    • Use the --setupinit parameter to install the agent as a service.
    • <path> is the installation path for the agent. This will be your <AGENT_HOME>[eg:/home/user/vuhealthagent].
  3. The install script will successfully install the agent and suggest how to start and stop the agent.

Update Existing Agent Configurations

If you already have agent installed on your target server and require a configuration update, please download the agent update package.

  1. Copy the downloaded vuHealthagent update package to and run the vuHealthagent update script.
    bash ./update <downloaded_tar_file>
  2. Restart the agent to apply the configuration changes.

Agent Operations

After the installation is complete, you can utilize the following commands to manage the agent.

  1. Start the agent
    <AGENT_HOME>/etc/rc.d/init.d/vuhealthagent start
  2. Stop the agent
    <AGENT_HOME>/etc/rc.d/init.d/vuhealthagent stop
  3. Status of the agent
    <AGENT_HOME>/etc/rc.d/init.d/vuhealthagent status

Uninstall the Agent

If you want to uninstall the agent on target server.

  1. Navigate into the vuHealthagent directory.
  2. Execute the vuHealthagent uninstall script.
    ./uninstall
